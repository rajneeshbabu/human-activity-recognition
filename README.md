Classify six daily activities from smartphone accelerometer and gyroscope data using machine learning. Built on the UCI HAR Dataset — recordings of 30 subjects wearing a Samsung Galaxy S II on the waist.

**Project Page:** [rajneeshbabu.github.io/human-activity-recognition](https://rajneeshbabu.github.io/human-activity-recognition/)

---

## Dataset

| Property | Value |
|----------|-------|
| Subjects | 30 volunteers, age 19–48 |
| Activities | WALKING · WALKING_UPSTAIRS · WALKING_DOWNSTAIRS · SITTING · STANDING · LAYING |
| Sensor | Samsung Galaxy S II (accelerometer + gyroscope) |
| Sampling rate | 50 Hz |
| Window | 2.56 s, 50% overlap (128 readings/window) |
| Features | 561 (time + frequency domain, pre-extracted) |
| Train / Test | 7,352 / 2,947 samples (70/30 split by subject) |

**Inertial signals available (raw, 128 timesteps × 9 channels):**
Body acceleration XYZ, Body gyroscope XYZ, Total acceleration XYZ

---

## Results

| Model | Test Accuracy |
|-------|--------------|
| Logistic Regression | **95.52%** |
| SVM (RBF, C=10) | **95.49%** |
| LightGBM | 93.45% |
| XGBoost | 93.42% |
| Random Forest | 92.57% |

**Key findings:**
- LAYING achieves 100% precision/recall — gravity signal is completely distinct
- SITTING vs STANDING is the hardest pair — both static, differ only in subtle posture angle
- Gravity features (`tGravityAcc-mean-XYZ`) are the most discriminative across all models
- Dynamic activities (walking variants) occasionally confused with each other (~5–8% error)
- 561 hand-crafted features make linear models (Logistic Regression, SVM) competitive with boosting

## Running

### 1. Train models

Open and run `har_notebook.ipynb` top to bottom. It will:

1. Load and explore the dataset (class distribution, PCA, t-SNE, raw signals)
2. Train 5 classifiers (Logistic Regression, SVM, Random Forest, XGBoost, LightGBM)
3. Evaluate and compare all models with confusion matrices and classification reports
4. Analyse feature importance and static vs dynamic activity separation
5. Save best models to `models/`

### 2. Run the Streamlit app

```bash
streamlit run app.py
```

Select any test sample, click **Predict Activity**, and see the SVM's prediction with per-class confidence bars.
